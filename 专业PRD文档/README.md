# PRD Arkham

> 把需求从一句话变成一份 PRD + 一个可交互原型，全程不超过 10 分钟。

PM 最怕两件事：需求没说清楚被研发打回来，和自己花几小时写完的文档没人看。PRD Arkham 就是为了解决这两件事而生的。

---

## 它能做什么

**你说，它写。** 口头描述、Word 文档、设计截图，任何形式的输入都能处理。

**写之前先问清楚。** 不会拿到需求就直接动笔——先输出结构化摘要，把所有有歧义的地方列出来让你确认，再开始写。你不用担心它自作主张。

**PRD + 原型一起产出。** 写完 PRD，自动生成对应的 HTML 交互原型。原型包含真实的空状态、加载态、错误路径，不只是摆好看的静态界面——研发打开就能直接参考。

**需求管理，不只是写文档。** 项目有多个系统？同一个页面被两个需求同时改？上线后要做 v2.0？这些都有对应的机制处理，不会越改越乱。

---

## 核心能力

**需求拆解确认** — 写 PRD 前输出结构化摘要，所有模糊点等你确认，绝不自行推断

**HTML 原型自动生成** — Tailwind CSS 单文件，含空状态 / 骨架屏 / 错误路径，浏览器直接打开

**微交互覆盖** — 按钮置灰条件、表单校验时机、弹窗优先级……每个有状态的组件都必须写完

**项目层级管理** — 市场 → 系统 → 模块 → 需求 → 版本，多系统并行不会乱

**增量版本 PRD** — v1.0 是完整基线，v2.0 只写变了什么，变更前后对照清晰可查

**交叉引用精度地图** — 同一页面被多个需求触碰时，自动声明边界、标注归属，杜绝互相覆盖

**APP 专项章节** — 自动识别 APP 需求，按需补充埋点整理、灰度发布节奏、分流逻辑、BI 数据提取

## 安装

### Claude Cowork / claude.ai

下载 [`prd-arkham.skill`](./prd-arkham.skill) → 点击「Copy to your skills」→ 重启对话

### Cursor

下载 [`prd-arkham.mdc`](./prd-arkham.mdc) → 放入 `.cursor/rules/` 文件夹

> `.mdc` 文件已完整内嵌所有 PRD 模板和原型生成指南，单文件开箱即用，无需额外配置。
