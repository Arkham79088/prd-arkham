# PRD Arkham

一个为产品经理设计的 AI Skill，将原始需求（口头描述、Word 文档、设计截图）标准化整理为结构化的 PRD、HTML 原型和索引体系。

适用于任何行业、任何市场、任何系统的产品需求管理。

---

## 功能亮点

- **全流程自动化**：需求输入 → 拆解确认 → PRD 编写 → HTML 原型生成 → 索引更新，一气呵成
- **强制拆解确认**：写 PRD 前必须先输出结构化摘要并等待用户确认，避免理解偏差返工
- **硬性规则保护**：时间节点必问（规则 A）、零推断原则（规则 B），杜绝自作主张
- **HTML 原型自动生成**：Tailwind CSS + Font Awesome，支持 Web 后台和移动端两种布局，含空状态、骨架屏、错误路径等完整状态
- **APP 需求专项**：识别 APP 端需求后自动询问埋点整理、灰度发布节奏、分流逻辑、BI 数据提取
- **增量版本管理**：v1.0 为完整基线，v2.0+ 只写差异，保留变更前后对照
- **项目层级管理**：市场 → 系统 → 模块 → 需求 → 版本，支持多市场多系统并行
- **微交互覆盖清单**：按需检查文本展示、按钮操作、表单输入、列表加载、弹窗、异常降级六个维度
- **细节深度原则**：每个有状态的组件必须覆盖所有有意义的状态，不只写主路径

## 安装方式

### Claude Cowork / claude.ai

1. 下载 [`prd-arkham.skill`](./prd-arkham.skill)
2. 在 Cowork 中点击该文件，选择「Copy to your skills」
3. 重启对话后即可使用

### Cursor

将 [`SKILL.md`](./SKILL.md) 的内容复制到你的 `.cursor/rules` 或项目根目录的 `.cursorrules` 文件中。

---

## 使用方式

安装后，直接描述你的需求、上传 Word 文档或设计截图，Skill 会自动检测环境并引导你完成整个 PRD 流程。

**首次使用**会引导你完成一次性配置（CodePen 账号、界面语言等）和项目初始化（市场/系统/目录结构）。

**日常使用**示例：
- 「帮我整理一下这个减免功能的需求」
- 「我上传了一份 Word，帮我转成 PRD」
- 「新建一个用户管理模块的需求」
